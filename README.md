# Go Monolith Boilerplate

This is a Go monolith boilerplate project for building a backend service. It includes a basic project structure, database migrations, and a development server with live reloading.

## Getting Started

To get started, you will need to have Go and `make` installed.

1.  **Clone the repository:**

    ```bash
    git clone https://github.com/Generalsimus/go-monolith-boilerplate
    cd go-monolith-boilerplate
    ```

2.  **Set up your environment variables:**

    Create a `.env` file in the root of the project and add the following:

    ```
    DATABASE_URL=postgres://user:password@localhost:5432/db_name?sslmode=disable
    ```

    Replace `user`, `password`, and `db_name` with your PostgreSQL connection details.

3.  **Run the database migrations:**

    ```bash
    make migrate-up
    ```

4.  **Start the development server:**

    ```bash
    make dev
    ```

    The server will start on `http://localhost:8080` and will automatically reload when you make changes to the code.

    *Note: API documentation is automatically generated by Huma and is available at `http://localhost:8080/docs` while in development mode.*

## Available `make` commands

*   `make dev`: Start the development server with live reloading.
*   `make migrate-new name=<migration_name>`: Create a new database migration.
*   `make migrate-up`: Apply all pending database migrations.
*   `make migrate-down`: Roll back the last database migration.
*   `make sqlc`: Generate Go code from your SQL queries.
*   `make db-update`: A convenience command that runs `migrate-up` and `sqlc` in one step.

## Technologies Used

*   [Go](https://golang.org/)
*   [PostgreSQL](https://www.postgresql.org/)
*   [chi](https://github.com/go-chi/chi) for routing
*   [pgx](https://github.com/jackc/pgx) for PostgreSQL driver and toolkit
*   [sqlc](https://github.com/sqlc-dev/sqlc) for generating type-safe Go code from SQL
*   [goose](https://github.com/pressly/goose) for database migrations
*   [air](https://github.com/cosmtrek/air) for live reloading
*   [websocket](https://github.com/gorilla/websocket) for WebSocket communication
*   [huma](https://github.com/danielgtaylor/huma) for API documentation and validation
